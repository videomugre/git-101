Git 101

por Fernando Osuna

Primero, ¿como moverse en el cmd?

cd ruta absoluta o relativa

cd.. (para moverse a la carpeta superior)

Tip: puedes pegar rutas en el cmd

Segundo, conceptos básicos

  • BRANCH ramificación de un proyecto.
  • MASTER BRANCH default.
  • COMMIT guardar los cambios en el repositorio local.
  • HEAD es el 'identificador' del COMMIT en el que el usuario se encuentra.
  • TAG identificación textual.
  • REPOSITORIO tiene toda la información referentes a los COMMITs.
  • PUSH sube los cambios locales al repositorio.
  • PULL descarga los cambios locales del repositorio.
  • CLONE copia por completo un repositorio.

Tercero, algo de configuración

git config --global user.name "Juan Perez"

git config --global user.email "juan.perez@clima-flex.com"

git config --global core.editor "notepad"

Empecemos. Una vez ubicados en la carpeta, inicializamos git

git init

Se crea la carpeta .git

En esa carpeta está todo lo que hacemos en git

En el archivo .gitignore se especifican las rutas o archivos que queremos git ignore

Agregamos todos los archivos para nuestro primer COMMIT

git add .

Para ver el estado de nuestro BRANCH en cualquier momento

git status

Nuestro primer COMMIT

git commit

Etiquetar nuestro COMMIT

git tag vX.Y.Z

git tag -a vX.Y.Z -m "Mensaje de TAG."

git describe --tags
(describe el tag actual)

Crear un nuevo BRANCH y entrar en él

git checkout -b nombre_del_branch

Tip: nombra tus BRANCHEs con el formato tipo/tema

Listar BRANCHEs

git branch

git branch --list "tipo/*"
(para listar por cierto nombre)

Para agregar los archivos nuevos al próximo COMMIT

git add -A

git add ruta/nombre_de_archivo
(para agregar archivos o rutas especificas)

Hacer COMMIT considerando los archivos modificados (y borrados) y escribir mensaje al mismo tiempo

git commit -a -m "Mensaje de COMMIT."

Para comparar 2 BRANCHEs

git diff branch1 branch2

Editar mensaje de último COMMIT

git commit --amend

Deshacer cambios de un archivo a última versión

git checkout HEAD -- nombre_de_archivo

Moverse entre BRANCHEs

git checkout nombre_del_branch

Guardar cambios sin hacer COMMIT

git stash -u

git stash pop
(los trae de vuelta listos para el COMMIT)

git stash list
(los lista)

git stash drop
(los borra)

(si nos movemos a otro branch sin hacer commit o stash, pasan cosas raras)

Combinar con otro BRANCH

git merge nombre_del_branch

¿Qué es un conflicto?

Cuando los 2 BRANCHEs a combinar tienen cambios diferentes

Se resuelven a mano

Borrar BRANCH

git branch -d nombre_del_branch

git branch -D nombre_del_branch
(si quieren ignorar cualquier advertencia)

Ver log de COMMITs

git log

(ENTER para avanzar y Q para salir)

Agregar repositorio remoto

git remote add origin nombre_de_repositorio

Subir COMMITs al repositorio

git push -u origin HEAD --tags

Bajar nuevos COMMITs del repositorio

git fetch

Bajar nuevos COMMITs del repositorio y hacer MERGE con ellos

git pull

Revertir el último COMMIT y considerar este mismo paso en su historia

git revert HEAD

git revert HEAD~n
(donde n es el número de COMMITS que quieres volver en su historia)

Resetea el proyecto a cierto COMMIT

git reset --soft HEAD
(deshace el último COMMIT pero conserva los cambios)

git reset --hard HEAD
(deshace completamente los cambios al último COMMIT)

git reset --hard origin/master
(se resetea con el master del repositorio)

Clonar proyecto

git clone ruta_a_repositorio
